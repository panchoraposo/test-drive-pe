== Bienvenido/a al workshop

En este workshop usarás **Red Hat Developer Hub** como un portal de autoservicio para construir, desplegar, asegurar y observar aplicaciones en OpenShift — potenciado con asistencia de IA mediante **Lightspeed** y el **MCP Gateway**.

=== Qué vas a aprender

* Usar **Software Templates** para crear servicios backend, frontend y MCP
* Observar **pipelines Tekton**, **GitOps con ArgoCD** y vistas de **Topology**
* Asegurar APIs con políticas de **Connectivity Link** (OIDC y API Key vía Kuadrant)
* Usar **OpenShift Lightspeed** con **MCP Gateway** para operaciones del cluster
* Monitorear la plataforma con **Grafana** y las pestañas **Graph** y **Traces** de **Service Mesh** en la consola OpenShift

=== Caso de negocio: Neuralbank

Neuralbank es una institución financiera modernizando su stack tecnológico. Como desarrollador/a, construirás tres componentes a través de Developer Hub:

[cols="1,3"]
|===
| Componente | Descripción

| **Customer Service MCP**
| Servidor MCP (Model Context Protocol) — expone operaciones del backend como herramientas invocables por IA

| **Neuralbank Backend**
| API REST en Quarkus para gestión de clientes y scoring de crédito

| **Neuralbank Frontend**
| SPA web conectada al backend y al MCP a través del gateway de Kuadrant
|===
