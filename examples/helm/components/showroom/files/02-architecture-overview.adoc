= Workshop Architecture Overview

This page describes the logical architecture of the Neuralbank scenario: how Developer Hub, GitOps, CI/CD, identity, API security, AI, and observability fit together on OpenShift.

== Component View

image::02-component-view.svg[Workshop component diagram, width=100%, link=_images/02-component-view.svg]

Each block has a clear role: the Hub orchestrates the experience; Git is the source of truth; ArgoCD applies the declarative model; Tekton automates the software supply chain; the gateway and policy layer exposes and protects traffic; Lightspeed adds AI assistance; and the observability stack monitors everything.

== Main Flow: From Template to Deployment

image::02-scaffolding-flow.svg[Scaffolding flow from Developer Hub to deployment, width=100%, link=_images/02-scaffolding-flow.svg]

. The developer opens **Developer Hub** and runs a **Software Template**.
. The template **publishes** a repository in **Gitea** with code, manifests, `devfile`, and pipeline definition.
. **ArgoCD** detects the Application and **synchronizes** manifests to the `{user_name}-neuralbank` namespace.
. **Tekton** runs a **PipelineRun** (clone, build, image, deploy) triggered by a **Gitea webhook**.
. OpenShift **schedules pods**, exposes **Services**, and creates **Gateway API** resources.
. The component is **registered** in the Developer Hub catalog.

== Connectivity Link Pattern

image::02-connectivity-link.svg[Connectivity Link pattern with Gateway API and Kuadrant, width=100%, link=_images/02-connectivity-link.svg]

The **Connectivity Link** pattern uses **Gateway API** with **Kuadrant** policies to secure exposed services:

* **Gateway** — traffic entry point (host, listeners, TLS)
* **HTTPRoute** — hostnames and routing rules to backend Services
* **AuthPolicy** — authentication via OIDC (Keycloak JWT) or API Key (X-API-Key header)
* **RateLimitPolicy** — rate limits to protect backends

=== AuthPolicy YAML Structure

[source,yaml,subs="attributes+"]
----
apiVersion: kuadrant.io/v1
kind: AuthPolicy
metadata:
  name: neuralbank-backend-auth
spec:
  targetRef:
    group: gateway.networking.k8s.io
    kind: HTTPRoute
    name: neuralbank-backend-route
  defaults:
    rules:
      authentication:
        api-key-auth:
          apiKey:
            allNamespaces: true
            selector:
              matchLabels:
                app: neuralbank-backend
                kuadrant.io/apikey: "true"
          credentials:
            customHeader:
              name: X-API-Key
          priority: 0
        oidc:
          jwt:
            issuerUrl: https://rhbk.{cluster_domain}/realms/neuralbank
          credentials:
            authorizationHeader:
              prefix: Bearer
          priority: 1
----

== Red Hat Connectivity Link MCP gateway (Technology Preview)

The link:https://docs.redhat.com/en/documentation/red_hat_connectivity_link/1.3/html/installing_the_mcp_gateway/mcp-gateway-install[Red Hat Connectivity Link MCP gateway] (Technology Preview) aggregates **Model Context Protocol** (MCP) servers behind a Gateway API endpoint. The operator is installed from the `redhat-operators` catalog (`preview` channel). OpenShift Lightspeed and other AI assistants use it to call cluster tools securely:

== Role of Each Component

[cols="1,3",options="header"]
|===
|Component |Role

|Developer Hub |Developer portal: catalog, templates, plugins, AI chat
|Gitea |Internal Git: source code, manifests, pipeline triggers
|ArgoCD |GitOps sync from Git to cluster state
|Tekton |Pipeline execution: clone, build, image, deploy
|Keycloak |Identity and SSO; feeds OIDC policies and Hub access
|Dev Spaces |Cloud IDE based on `devfile`, connected to the same repo
|Gateway API / Kuadrant |Traffic ingress, routing, and auth/rate-limit policies
|OpenShift Lightspeed |AI assistant with MCP Gateway for cluster operations
|LiteLLM |LLM proxy for Developer Lightspeed
|Grafana |Monitoring dashboards and alerting
|OpenShift Service Mesh |Traffic graph and distributed traces in the web console
|Mailpit |Email testing for notifications
|===
